🏥 HealthChain: Decentralized EHR SystemHealthChain is a patient-centric, decentralized Electronic Health Record (EHR) platform. It leverages blockchain technology to return data ownership to patients, ensuring medical records are immutable, secure, and accessible only to authorized healthcare providers.🚀 Live Demo: https://healthchain-frontend.vercel.app🌟 Key FeaturesAbsolute Patient Ownership: Patients have the sole authority to grant or revoke access to their medical records for doctors and diagnostic centers.Immutable Audit Trail: All access permissions and record additions are anchored to the Ethereum Sepolia blockchain, creating a permanent, tamper-proof history.Decentralized Storage: Medical files are stored on IPFS (via Pinata), ensuring no single point of failure and high data availability.Hybrid Architecture: Combines the speed of MongoDB Atlas for metadata with the security of Blockchain for integrity.Real-time Feedback: Integrated Transaction Stepper provides users with live status updates during blockchain mining and IPFS pinning.🛠️ Tech StackLayerTechnologyFrontendReact.js, Tailwind CSS, Framer Motion, Lucide IconsBackendNode.js, Express.jsBlockchainSolidity, Ethereum (Sepolia Testnet), Ethers.jsDatabaseMongoDB Atlas (User Auth & Metadata)StorageIPFS (via Pinata)DeploymentVercel (Frontend), Koyeb (Backend)📐 System ArchitectureHealthChain uses a "Hybrid Web3" approach to balance cost and performance.Identity: Users register via the Node.js backend, which hashes passwords with Bcrypt and stores profiles in MongoDB.Permissions: Patients interact with the Smart Contract to update access mappings on-chain.Storage: Professionals upload files; the backend streams these to IPFS and receives a CID (Content Identifier).Linking: The CID is stored on the blockchain, linked to the patient's unique ID.🚀 Getting StartedPrerequisitesNode.js (v16+)Alchemy/Infura API Key (for Sepolia RPC)Pinata API Keys (for IPFS)Metamask Wallet (with Sepolia test ETH)InstallationClone the repository:Bashgit clone https://github.com/your-username/healthchain.git

🔄 WorkflowStep 1: Patient registers and grants access to a Doctor ID.Step 2: Doctor searches for Patient ID; the system verifies the permission on the Blockchain.Step 3: Doctor uploads a lab report.
